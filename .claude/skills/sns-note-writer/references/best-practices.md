# note ベストプラクティス

## プラットフォーム特性

- クリエイター向け長文プラットフォーム
- 「スキ」（いいね）ベースのエンゲージメント
- 有料記事で収益化可能
- 技術系よりも体験談・考察が好まれる

## コンテンツ方針

### 記事タイプ

| カテゴリ | 説明 | 例 |
|---------|------|-----|
| experience | 体験談・実践記録 | 「AIエージェントで自動化した1ヶ月の記録」 |
| insight | 考察・意見 | 「なぜ Claude Code を選んだのか」 |
| tutorial | やさしいチュートリアル | 「非エンジニアのための AI 活用入門」 |
| review | レビュー・感想 | 「話題の AI ツールを全部試してみた」 |

### Zenn/Qiita との差別化

- Zenn/Qiita: コード主体、技術的に正確
- note: **体験談・考察として再構成**
  - 「なぜその技術を選んだか」
  - 「開発の裏話」
  - 「学んだこと」
  - 「失敗から得た教訓」

## 収益化戦略

### 前半無料 + 後半有料

```
[無料部分]
- 導入: 問題提起・共感
- 概要: 何をしたか
- ハイライト: 最も面白い結果

[有料部分]（100-500円）
- 詳細な手順
- 具体的な数値・データ
- テンプレート・チェックリスト
- 失敗パターンと対策
```

## 投稿スケジュール

| 時間帯 | エンゲージメント | 適した内容 |
|--------|----------------|-----------|
| 朝 8-9時 | 高い | 通勤中に読める記事 |
| 昼 12-13時 | 高い | 昼休みの読み物 |
| 夜 21-22時 | 最も高い | じっくり読む長文 |

**投稿直後2時間の「スキ」が結果を左右する → 最適時間帯に投稿**

## 本文構成ガイドライン

1. **フック** — 読者の共感を得る問いかけや体験
2. **背景** — なぜこれをやろうと思ったか
3. **本論** — 体験の詳細（時系列が効果的）
4. **気づき** — やってみてわかったこと
5. **まとめ** — 読者へのメッセージ

## 品質基準

- 文字数: 2,000-5,000文字
- 画像: アイキャッチ必須 + 本文中2-3枚
- H2/H3 で区切り、スキャンしやすい構成
- 個人的な視点・感情を入れる（「思った」「感じた」「驚いた」）
- 技術用語は平易に言い換える
