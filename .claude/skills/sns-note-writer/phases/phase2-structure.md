# Phase 2: 構成設計

## 前提条件

- `work/strategy.json` が存在すること（Phase 1 の出力）

## 目的

strategy.json を入力に、note 記事のストーリー構成と有料分割ポイントを設計する。

## 手順

### Step 1: ストーリー構成設計

体験ベースのストーリー構成を設計:

- 起承転結またはフック → 体験 → 学び → 提案
- H2/H3 の見出し構成（3-5セクション）
- 読者の共感ポイントを各セクションに配置

### Step 2: 有料分割ポイント設計

前半無料 + 後半有料の分割ポイントを設計:

- 分割位置: 全体の60-70%地点（価値の核心に入る直前）
- 無料部分で十分な価値を提供し、有料部分への期待を作る
- 有料部分には具体的な手順・数値・テンプレートを配置

### Step 3: フック設計

- 共感を呼ぶ導入（「〜ということありませんか？」）
- 結果を先に見せる（「結果、〜になりました」）

### Step 4: 構成 JSON 出力

設計結果を `work/structure.json` に保存。

## 入力

- ファイル: `work/strategy.json`

## 出力

- ファイル: `work/structure.json`
- スキーマ: `schemas/structure.schema.json`

## バリデーション

- `sections` が 3-5 個であること
- `paidBreakpoint`（有料分割ポイント）が設定されていること
