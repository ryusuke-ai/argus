# Phase 1: リサーチ & 戦略

## 前提条件

- ユーザーからテーマ・トピックが提供されていること

## 目的

note のトレンドを調査し、体験・ストーリー系の記事戦略を策定する。

## 手順

### Step 1: トレンド調査

WebSearch で以下を調査:

- テーマに関する note での人気記事
- 検索キーワード: `site:note.com {テーマ}`
- note のトレンドカテゴリの傾向

### Step 2: 体験角度の探索

- 技術的な話題を「体験・ストーリー」として再構成する角度を探る
- 「やってみた」「失敗した」「学んだ」系のフレーミング
- 読者が共感できるポイントの特定

### Step 3: 差別化ポイント決定

- Qiita/Zenn のコード主体記事との差別化
- 個人的な視点・感想を重視
- ターゲット読者: 非エンジニアも含む幅広い層
- カテゴリ決定: experience / insight / tutorial / review

### Step 4: 戦略 JSON 出力

調査結果を `work/strategy.json` に保存。

## 入力

- ユーザーのテーマ・トピック（テキスト）

## 出力

- ファイル: `work/strategy.json`
- スキーマ: `schemas/strategy.schema.json`

## バリデーション

- `storyAngle` が設定されていること
- `targetEmotion`（読者に感じてほしい感情）が設定されていること
