# Phase 4: 3秒フック・ループ最適化 & 仕上げ

## 前提条件

- 前フェーズ (content) の JSON 出力が入力として提供されること

## 目的

content フェーズの出力を入力に、TikTok 固有の品質チェックを行い最終成果物を JSON で出力する。

## 手順

### Step 1: 3秒フック厳守チェック

- 最初の3秒以内にフックがあるか
- 冒頭でのダラダラ挨拶がないか
- スワイプされる前に「見続ける理由」を提示しているか

### Step 2: ループ構成確認

- 最後が最初に繋がるか
- 再視聴率を高める仕掛けがあるか

### Step 3: 縦型構成確認

- 重要要素が画面中央に配置されているか
- テキストオーバーレイがミュートでも伝わるか

### Step 4: 禁止事項チェック

- 横型動画指示がないこと
- 冒頭ダラダラ挨拶がないこと
- 3分超の指示がないこと
- テキストオーバーレイ指示があること

### Step 5: 投稿時間推奨

| 時間帯 (JST)    | エンゲージメント |
| --------------- | ---------------- |
| 7:00-8:00       | 高い（朝の通勤） |
| 17:00-18:00     | 高い（帰宅中）   |
| **20:00-22:00** | **最高**         |

## 出力形式

以下の JSON オブジェクトを出力すること（ファイル保存は不要）:

```json
{
  "title": "動画タイトル",
  "description": "キャプション",
  "script": "台本テキスト",
  "textOverlays": ["テキスト1", "テキスト2"],
  "soundSuggestion": "サウンド提案",
  "metadata": {
    "category": "tutorial",
    "estimatedDuration": "30秒",
    "hashtags": ["#AI", "#プログラミング"],
    "recommendedPostTime": "20:00-22:00"
  }
}
```

## バリデーション

- 動画が60秒以内であること
- 3秒以内にフックがあること
- テキストオーバーレイが含まれていること
- hashtags が適切に設定されていること
