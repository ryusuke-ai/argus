# Phase 4: 3秒フック・ループ最適化 & 仕上げ

## 前提条件

- `work/content.json` が存在すること（Phase 3 の出力）

## 目的

content.json を入力に、TikTok 固有の品質チェックを行い最終出力する。

## 手順

### Step 1: 3秒フック厳守チェック

- 最初の3秒以内にフックがあるか
- 冒頭でのダラダラ挨拶がないか
- スワイプされる前に「見続ける理由」を提示しているか

### Step 2: ループ構成確認

- 最後が最初に繋がるか
- 再視聴率を高める仕掛けがあるか

### Step 3: 縦型構成確認

- 重要要素が画面中央に配置されているか
- テキストオーバーレイがミュートでも伝わるか

### Step 4: 禁止事項チェック

| チェック項目 | OK/NG |
|------------|-------|
| 横型動画指示がないか | |
| 冒頭ダラダラ挨拶がないか | |
| 3分超の指示がないか | |
| テキストオーバーレイ指示があるか | |

### Step 5: 投稿時間推奨

| 時間帯 (JST) | エンゲージメント |
|--------------|---------------|
| 7:00-8:00 | 高い（朝の通勤） |
| 17:00-18:00 | 高い（帰宅中） |
| **20:00-22:00** | **最高** |

### Step 6: 最終出力

- `output/script.json`: 台本 + メタデータ（台本テキスト、テキストオーバーレイ、キャプション、サウンド提案）

### Step 7: ユーザー承認（BLOCKER）

`AskUserQuestion` で台本サマリーを提示して承認を得る。

## 入力

- ファイル: `work/content.json`

## 出力

- ファイル: `output/script.json`
