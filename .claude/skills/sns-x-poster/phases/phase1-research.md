# Phase 1: リサーチ & 戦略

## 前提条件

- ユーザーからテーマ・トピックが提供されていること

## 目的

X 上のトレンドを確認し、最適な投稿角度・タイプを決定する。

## 手順

### Step 1: トレンド確認

WebSearch で以下を調査:

- テーマに関する X 上の最新の話題
- 検索キーワード: `{テーマ} site:x.com` または `{テーマ} Twitter`
- 同ジャンルの人気投稿の傾向（いいね数、リプライ数）

### Step 2: 角度決定

投稿カテゴリを決定:

| カテゴリ     | テンプレート型      |
| ------------ | ------------------- |
| `news`       | 速報型              |
| `experience` | 体験談型 / 失敗談型 |
| `tips`       | 手順型 / 証拠付き型 |
| `code`       | コード型 / 比較型   |
| `summary`    | 証拠付き型          |
| `discussion` | 問いかけ型          |

### Step 3: 投稿タイプ判定

- **single**: 独立した1投稿で完結する場合
- **thread**: 情報量が多く、3-6投稿に分割すべき場合

### Step 4: 戦略 JSON 出力

調査結果を `work/strategy.json` に保存。

## 入力

- ユーザーのテーマ・トピック（テキスト）

## 出力

- ファイル: `work/strategy.json`
- スキーマ: `schemas/strategy.schema.json`

## バリデーション

- `postType` が "single" or "thread" で設定されていること
- `category` が設定されていること
- `angle`（投稿の角度）が設定されていること
