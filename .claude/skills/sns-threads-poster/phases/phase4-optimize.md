# Phase 4: プラットフォーム最適化 & 仕上げ

## 前提条件

- `work/content.json` が存在すること（Phase 3 の出力）

## 目的

content.json を入力に、Threads 固有のチェックを行い最終出力する。

## 手順

### Step 1: 外部リンクチェック

- メイン投稿に外部リンクが含まれていないことを確認
- URL が含まれていたら即座に削除

### Step 2: ハッシュタグチェック

- ハッシュタグが0-1個であること
- Threads ではハッシュタグの効果が薄いため基本的に不要

### Step 3: 質問チェック

- 質問や問いかけで終わっているか確認
- 投稿の70%以上に質問があること

### Step 4: クロスポストチェック

- X の投稿と同一内容になっていないか
- Threads 独自のトーン・角度になっているか

### Step 5: 投稿時間推奨

| 時間帯 (JST)    | エンゲージメント           |
| --------------- | -------------------------- |
| 7:00-8:00       | 高い（朝の確認）           |
| 12:00-13:00     | 高い（昼休み）             |
| **20:00-21:00** | **最高**（夜のリラックス） |

### Step 6: 最終出力

- `output/post.json`: Threads 投稿データ

### Step 7: ユーザー承認（BLOCKER）

`AskUserQuestion` で承認を得る。

## 入力

- ファイル: `work/content.json`

## 出力

- ファイル: `output/post.json`
