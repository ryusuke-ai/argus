# Zenn ベストプラクティス

## プラットフォーム特性

- 技術記事に特化した日本語プラットフォーム
- GitHub 連携で記事管理（articles/ ディレクトリに Markdown）
- 「本」機能で体系的知識を有料販売可能
- トレンド入りで自然流入が大幅増加

## コンテンツ方針

### バズりやすい記事タイプ

1. **ファーストインプレッション** — 新ツール/ライブラリの速報記事（最もバズる）
2. **チュートリアル** — 手順付きの実践ガイド
3. **比較記事** — A vs B の実測データ付き比較
4. **ディープダイブ** — 内部実装や仕組みの解説

### 記事構成

- タイトルに技術名を必ず含める
- コードブロックは**実行可能な完全版**を掲載
- リリース直後の速報 → 翌日深掘り → 週末まとめ の3段構成が効果的

## 投稿スケジュール

| 時間帯 | エンゲージメント | 適した内容 |
|--------|----------------|-----------|
| 平日朝 7-9時 | 高い | チュートリアル、速報 |
| 平日昼 12-13時 | 高い | Tips、比較記事 |
| 日曜夜 20-22時 | 高い | ディープダイブ、まとめ |

## フロントマター仕様

```yaml
---
title: "記事タイトル（技術名 + 具体的内容）"
emoji: "🤖"          # アイキャッチ用絵文字（必須）
type: "tech"         # "tech" or "idea"
topics: ["claudecode", "typescript", "ai"]  # 最大5個
published: true      # false = 下書き
---
```

## スラッグ規則

- 12-50文字
- 小文字英数字とハイフンのみ (`a-z0-9-`)
- 例: `claude-code-agent-sdk-guide`

## タグ選定ガイドライン

| 優先度 | タグ例 | 理由 |
|--------|--------|------|
| 必須 | メイン技術名 (claudecode, typescript) | 検索流入 |
| 推奨 | カテゴリ (ai, automation) | トレンドページ表示 |
| 任意 | 関連技術 (nodejs, react) | 関連記事からの流入 |

## 品質基準

- 文字数: 3,000-10,000文字（短すぎると価値が薄い、長すぎると読了率低下）
- コードブロック: 3個以上推奨
- 見出し: H2/H3 で論理的に区切る
- 画像: スクリーンショットや図解を適宜挿入
