## 1. ディスカバリーの仕組み

### 1.1 Spotify のランキングアルゴリズム

Spotify のポッドキャストチャートは毎日更新され、以下のエンゲージメント指標を総合的に評価する。

| シグナル         | 説明                               | 重み     |
| ---------------- | ---------------------------------- | -------- |
| フォロー数       | 番組をフォローしたリスナーの総数   | **最高** |
| フォロー急増     | 短期間でのフォロー数の伸び         | **最高** |
| 再生数（Plays）  | エピソードが60秒以上再生された回数 | 高       |
| ユニークリスナー | 個別のリスナー数                   | 高       |
| 完了率           | エピソードをどこまで聴いたか       | 高       |
| リピートリスナー | 毎週戻ってくるリスナーの割合       | 中-高    |
| リスナー成長率   | 期間あたりの新規リスナー増加速度   | 中       |

**核心**: ダウンロード数だけでは不十分。エンゲージメント（完了率、リピート視聴、フォロー）が伴わないダウンロードはチャート上昇に寄与しにくい。

#### Spotify の「Plays」指標（2025年5月導入）

2025年5月、Spotify は新指標「Plays」を導入。**エピソードが60秒以上再生された回数**をカウントする。音声・動画の両方を包含し、50K再生を超えるとマイルストーン表示になる（クリエイターダッシュボードでは正確な数値を引き続き確認可能）。

#### Spotify のディスカバリー機能（2025年更新）

- **ポッドキャスト推薦フィード**: ホーム画面のショートカット下部に新設。なぜおすすめなのかの説明付き
- **Podcasts Following フィード**: フォロー中の番組の新エピソードを Podcast タブに一元集約
- **「In this episode」セクション**: エピソード内から他コンテンツへリンク可能
- **コメント機能**: リスナーとの直接対話が可能に

## 2. エピソードパラメータ

### 2.1 エピソードの最適尺

データに基づく推奨尺:

| フォーマット       | 推奨尺  | 完了率への影響                               |
| ------------------ | ------- | -------------------------------------------- |
| 1人語り（ソロ）    | 15-30分 | 18-22分が完了率最高ゾーン（セッションの41%） |
| 対談・インタビュー | 30-60分 | 45分を超えると完了率が低下しやすい           |
| ニュースまとめ     | 10-20分 | 短くまとめるほど完了率が上がる               |
| チュートリアル     | 20-30分 | 手順の複雑さに応じて調整                     |
| ディープダイブ     | 25-45分 | 内容が十分なら長めでも完了率を維持可能       |

**核心**: **18-22分帯が全体セッションの41%を占め、完了率が最も高い**。28-32分帯は完了率が約9ポイント低下する。「コンテンツが必要とする長さにする。1分たりとも余計にしない」が鉄則。

**業界平均完了率**: 全体平均は57.9%。**70%以上なら優秀**。メキシコ62.4%、英語圏54.2%と地域差あり。

### 2.2 配信時間帯

#### グローバルの傾向

| 曜日      | 推奨度 | 理由                                       |
| --------- | ------ | ------------------------------------------ |
| 火曜-木曜 | **高** | 木曜がダウンロード数最多。通勤時間帯に合致 |
| 月曜      | 中-高  | 週の始まりでリスナーの注意を獲得しやすい   |
| 金曜-日曜 | 低-中  | 週末はリスニング時間が減少傾向             |

#### 時間帯の最適化

| 時間帯（JST） | 推奨度 | 理由                                                   |
| ------------- | ------ | ------------------------------------------------------ |
| 5:00-7:00     | **高** | 朝の通勤前に配信完了。アプリに新エピソードが並ぶ       |
| 7:00-9:00     | 高     | 通勤時間帯。リスナーがポッドキャストアプリを開くピーク |
| 12:00-13:00   | 中     | 昼休みのリスニング                                     |
| 21:00-23:00   | 中     | 夜のリラックスタイム                                   |

**推奨**: **月曜 5:00-7:00 JST に配信**。朝の通勤時間帯にリスナーのフィードに新エピソードが並ぶようにする。一貫性が最も重要であり、同じ曜日・時間に配信し続けることでリスナーの習慣を形成する。

#### 日本市場の特性

- 日本のポッドキャスト月間リスナーは約200万人以上（2025年時点）
- 通勤時間帯（7:00-9:00、17:00-19:00）がリスニングのゴールデンタイム
- スマートスピーカー普及により自宅でのリスニングも増加傾向
- ジャパンポッドキャストアワードの開催により認知度が向上中

### 2.3 タイトルの最適化

#### 効果的なタイトルパターン

| パターン           | 例                                                 | 効果               |

| テーマ + 疑問      | 「AI エージェントの権限設計 --- なぜ分離が必要か」 | 知的好奇心を刺激   |
| 数字 + 結果        | 「Claude Code で開発速度が3倍になった設定5つ」     | 具体的な価値を提示 |

## 3. コンテンツフォーマット別パフォーマンス

### 3.1 1人語り vs 対談

| フォーマット | メリット                                               | デメリット                       | 推奨尺  |
| ------------ | ------------------------------------------------------ | -------------------------------- | ------- |
| 1人語り      | 制作スピードが速い、一貫した品質、スケジュール管理容易 | トーンが単調になりやすい         | 15-30分 |
| 対談         | 自然な会話感、ゲストの聴衆流入、多角的視点             | スケジュール調整、音質のばらつき | 30-60分 |
| パネル       | 多様な意見、盛り上がりやすい                           | 収録が複雑、編集コスト高         | 45-90分 |

**AI/テック系での推奨**: **1人語りをベースに、月1-2回ゲスト対談**。1人語りは制作効率が高く、一貫したスケジュールを維持しやすい。ゲスト回はクロスプロモーションによる新規リスナー獲得に活用。

### 3.2 冒頭フックの重要性

リスナー離脱は**最初の30秒-1分で決まる**。イントロが90秒を超えるとドロップオフ率がほぼ2倍になる。

#### 効果的なフック構造

| フック手法   | 例                                                               | 効果             |
| ------------ | ---------------------------------------------------------------- | ---------------- |
| 結果の先出し | 「今日の話を聴けば、AI エージェントの権限設計で事故を防げます」  | 価値の即時提示   |
| 衝撃的事実   | 「先週、AI に本番 DB を全削除されました」                        | 注意の即時獲得   |
| 問いかけ     | 「あなたの AI エージェント、本番環境で何でもできる状態ですか？」 | 自分事化         |
| ティーザー   | 「後半で、この問題を一発で解決した設定を共有します」             | 最後まで聴く動機 |

**推奨構造**: 冒頭20-40語で「なぜ聴くべきか」を伝え、30-60秒以内にテーマに入る。

### 3.3 チャプターマーカーの効果

チャプターマーカーはリスナー体験を大幅に向上させる:

- **完了率向上**: 興味のあるセクションに直接ジャンプ可能で、途中離脱を防ぐ
- **リピート視聴促進**: 特定のチャプターを再度聴くために戻るリスナーが増加
- **Apple Podcasts 連携**: 2025年のアップデートで自動チャプター生成 + タイムドリンク機能が追加
- **検索性向上**: チャプタータイトルが検索対象に含まれる

### 3.4 配信スケジュールの一貫性

**週1回の一貫した配信を6ヶ月維持した番組は、不定期配信の番組と比較して320%の聴衆成長**を達成する。

| 配信頻度  | メリット                       | デメリット           | 推奨対象           |
| --------- | ------------------------------ | -------------------- | ------------------ |
| 毎日      | 急速なカタログ構築             | バーンアウトリスク高 | ニュース系のみ     |

| 隔週      | 制作負荷が低い                 | リスナーが忘れやすい | 長尺コンテンツ     |

## 5. やってはいけないこと

### 5.1 配信抑制リスク（ディスカバリー低下）

以下の行為はプラットフォームからの推薦が抑制され、ディスカバリーが著しく低下する。

| 行為                       | リスク                                         | 対策                           |
| -------------------------- | ---------------------------------------------- | ------------------------------ |
| 不定期配信                 | アルゴリズムが「活動停止」と判断、推薦頻度低下 | 固定スケジュール厳守           |
| タイトルの釣り             | クリック率は上がるが完了率低下→推薦抑制        | タイトルと内容を一致させる     |
| メタデータのキーワード詰め | スパム判定されカテゴリランキングから除外       | キーワードは自然に配置         |
| 人為的なダウンロード操作   | チャート操作検知でアカウント停止リスク         | オーガニック成長を目指す       |
| ガイドライン違反コンテンツ | 即座に配信停止、アカウント停止の可能性         | コミュニティガイドラインを遵守 |

### 5.2 リスナー離脱を招く行為

| 行為                       | データ/根拠                                         | 改善策                              |
| -------------------------- | --------------------------------------------------- | ----------------------------------- |
| 90秒超のイントロ           | ドロップオフ率がほぼ2倍に増加                       | 30-60秒で本題に入る                 |
| 低品質な音声               | リスナーが即座に離脱する最大要因の1つ               | 192kbps以上、ノイズリダクション必須 |
| 宣伝の唐突な挿入           | 自然に統合された広告と比較して35%多くスキップされる | コンテンツに溶け込む形で挿入        |
| テーマからの脱線           | リスナーの期待を裏切り、信頼を損なう                | 収録前に詳細なアウトラインを準備    |
| 具体例のない抽象論         | 「〜が大事です」だけでは響かない                    | 必ず実体験・データ・具体例を添える  |
| 前提知識が必要なコンテンツ | 新規リスナーが理解できず離脱                        | 初回リスナーでもわかる導入を入れる  |

### 5.3 成長を阻害する行為

| 行為                   | 影響                                      | 改善策                      |
| ---------------------- | ----------------------------------------- | --------------------------- |
| ショーノートを書かない | 検索性ゼロ、新規リスナーが見つけられない  | 300語以上の詳細ショーノート |
| チャプターマーカーなし | 長尺エピソードの完了率低下                | 3-5個のチャプターを設定     |
| SNSでの告知をしない    | 新規リスナーの主要流入経路を失う          | X、Threads、YouTube で告知  |
| レビュー依頼をしない   | Apple Podcasts のランキングシグナルを逃す | CTA でレビュー・評価を依頼  |
| 収益化に偏りすぎる     | コンテンツの質低下→リスナー離脱           | リスナー価値を最優先        |

## 6. 配信・自動化のルール

### 6.1 RSS フィードの最適化

RSS フィードはポッドキャストの基盤。各プラットフォームへの配信品質を左右する。

| 最適化項目             | 推奨設定                                                 |
| ---------------------- | -------------------------------------------------------- |
| フィードバリデーション | Cast Feed Validator で定期検証                           |
| 番組タイトル           | プライマリキーワードを含む。60文字以内                   |
| 番組説明文             | 先頭125文字にキーワード配置。全体で400-600文字           |
| エピソード説明         | キーワードを5回以上自然に含む。300語以上                 |
| カバーアート           | 3000x3000px、RGB、JPEG/PNG。モバイルで視認できるデザイン |
| カテゴリ               | プライマリ + セカンダリの2つを設定                       |
| 言語タグ               | `ja` を明示的に設定                                      |
| エンクロージャ         | ファイルサイズ、MIME タイプ、duration を正確に記載       |

#### RSS フィードの技術的チェック項目

- `<itunes:image>` タグでカバーアートを指定（3000x3000px 推奨）
- `<itunes:category>` で適切なカテゴリを設定
- `<itunes:explicit>` タグを含める（false でも明示が必要）
- `<podcast:transcript>` タグでトランスクリプトファイルをリンク
- `<podcast:chapters>` タグで JSON チャプターファイルをリンク
- エピソードの `<enclosure>` に正確な `length`（バイト数）を含める

### 6.2 音質の基準

| パラメータ     | 推奨値                                 | 備考                                     |
| -------------- | -------------------------------------- | ---------------------------------------- |
| フォーマット   | MP3                                    | 最も互換性が高い                         |
| ビットレート   | **128-192kbps**（モノラル）            | 音声のみなら128kbps、音楽含むなら192kbps |
| サンプルレート | **44.1kHz**                            | CD標準。48kHzは動画向け                  |
| チャンネル     | モノラル推奨                           | 1人語りはモノラル。対談はステレオも可    |
| 収録時         | 24bit / 48kHz                          | 編集余地を確保するため高品質で収録       |
| ノイズ除去     | 必須                                   | 背景ノイズ、リップノイズ等を除去         |
| ラウドネス     | -16 LUFS（モノ）/ -19 LUFS（ステレオ） | Apple Podcasts 推奨値                    |

**Apple Podcasts の公式要件**:

- サンプルレート: 44.1kHz
- ビットレート: 推奨128kbps以上
- フォーマット: MP3, M4A, MOV（動画）
- ファイルサイズ: 可能な限り小さく（ダウンロード速度に影響）
