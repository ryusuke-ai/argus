# Qiita ベストプラクティス

## プラットフォーム特性

- 日本最大の技術記事共有プラットフォーム
- LGTM（いいね）ベースのランキングシステム
- トレンド入りで大幅な流入増加
- 初心者向けAI記事が最もバズりやすい

## コンテンツ方針

### バズりやすい記事タイプ

1. **初心者向けチュートリアル** — 「○○入門」「○○の始め方」
2. **比較記事** — 「A vs B」の実測データ付き
3. **Tips集** — 「○○で使える便利技N選」
4. **How-to** — 具体的な手順付きガイド

### トレンド入り戦略

- LGTM 15 でデイリートレンド入り
- X でシェアして初速の LGTM を稼ぐ
- 投稿直後2時間が勝負

## タイトル設計

- **20-36文字**（長すぎると切れる）
- ターゲットレベルを明示（「初心者向け」「実践」等）
- 技術名を必ず含める
- 具体的な数値や成果を入れる

**良い例:**
- 「【初心者向け】Claude Code で TDD を始める3ステップ」
- 「TypeScript 5.x の新機能まとめ（実コード付き）」

**悪い例:**
- 「最近やってみたこと」（抽象的すぎる）
- 「Claude Code について思うこと」（具体性がない）

## タグ設計

- 最大5個
- Qiita 形式: `[{name: "ClaudeCode", versions: []}, ...]`
- メイン技術 + 関連技術 + カテゴリ

| 優先度 | タグ例 | 理由 |
|--------|--------|------|
| 必須 | メイン技術名 | 検索・フィード表示 |
| 推奨 | 言語/FW名 | 技術スタック明示 |
| 推奨 | カテゴリタグ (AI, 初心者) | ターゲット明示 |

## 投稿スケジュール

| 時間帯 | エンゲージメント | 適した内容 |
|--------|----------------|-----------|
| 月曜朝 7-8時 | 最も高い | 週初めの学習記事 |
| 平日昼 12-13時 | 高い | 昼休み読み切りサイズ |
| 平日夕方 18-19時 | 高い | 退勤後の技術記事 |

## 本文構成ガイドライン

1. **導入** — 何が学べるか、前提知識は何か
2. **環境構築** — 最小限の手順で動く状態に
3. **本論** — コード例付きで実践的に
4. **応用** — 発展的な使い方
5. **まとめ** — 要点の振り返り

## 品質基準

- 文字数: 5,000-15,000文字（できれば1万文字程度）
- コードブロック: 実行可能な完全版を掲載
- 環境構築のみで終わらない（実用的な成果まで含める）
- スクリーンショット/図解を適宜挿入
